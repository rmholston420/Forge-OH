# Forge-OH Start Here

Read `workflow-notes.md` first before using any other planning, implementation, or build document in this workspace. That workflow document is the controlling operating guide for how all changes must be proposed, inspected, written, verified, and pushed.

## Mandatory first read

1. Read `workflow-notes.md` first.
2. Follow its user workflow contract before proposing any repo change.
3. Use its preferred write pattern for insertion scripts.
4. Use its inspection and verification patterns before and after every modification.

## Document order

- First: `workflow-notes.md`
- Second: `Forge-OH Build Plan — Definitive.md` or `Forge-OH-Build-Plan-Definitive.md` when implementation scope, architecture, or slice sequencing is needed.
- Third: the narrower implementation specs only after the workflow notes and definitive build plan have been checked.

## Why this exists

This workspace contains multiple large planning and specification documents. Without a single first-read entry point, later work can drift away from the required operating workflow, especially around paste-safe scripting, no manual editing, inspection-first changes, and push-to-main expectations.

## Non-negotiable reminder

If any other document conflicts with `workflow-notes.md` on process, scripting style, inspection order, or user interaction pattern, `workflow-notes.md` wins.
